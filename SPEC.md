# SPEC.md — 异想天开（YiXiang TianKai）产品规格

## 一、产品愿景

把 OpenClaw 那种"AI Agent 自由控制本地环境"的工作流，**封装成一个面向终端用户的 Windows 桌面应用**。

普通用户：
- 不用搭服务、不用 Claude / Cursor 订阅、不用配 API Key
- 下载安装包，注册登录 → 立刻有一个可以"自己读写文件、跑命令、控浏览器"的 AI 工作台
- 用多少花多少（按积分扣 token），到期或额度不足自动停服

管理员（你）：
- 单独的 Web 后台，集中管理用户：账号、密码、积分、可用模型、有效期
- 一处管控全部上游 LLM Key（OpenAI / Claude / 方舟 / 通义 / DeepSeek …）
- 所有 LLM 请求经 blog 中转，安全 + 可审计 + 可计费

## 二、用户场景速写

**场景 A：小张装好"异想天开" 1.0**
1. 双击 MSI 装好，登录页输入账号密码（你提前创建）
2. 进入工作台，看到聊天框 + 文件树 + 终端 Tab
3. 输入"帮我把 D:\新员工资料 里所有 PDF 提取出名字邮箱，整理成 Excel"
4. Agent 开始：列目录 → 调 LLM 制定计划 → 调本地 PowerShell 跑脚本 → 用 OCR / 文本提取 → 输出 .xlsx
5. 任务完成，扣 12 积分（约 12000 tokens），剩余 488 积分

**场景 B：管理员（你）登录后台**
- 浏览器打开 `https://blog.tczeng.top/ai-app_admin`
- 用户列表 → "张三" → 充值 500 积分 / 改密码 / 把可用模型从 `方舟-doubao` 扩展到 `claude-3.5-sonnet` / 设到期日 2026-12-31

## 三、功能模块清单

### 客户端（Tauri）
- **F-CLI-1** 登录 / 注销 / 自动续期 token
- **F-CLI-2** 工作台主界面（侧栏 + 聊天主区 + 工具面板）
- **F-CLI-3** 聊天交互：流式输出、Markdown 渲染、代码高亮、附件上传
- **F-CLI-4** 多会话管理（左侧会话列表 + 新建/重命名/删除）
- **F-CLI-5** 本地工具：
  - 文件读写 / 列目录 / 增删改 / 搜索
  - PowerShell / cmd 执行（带审批门）
  - 浏览器控制（基于 CDP，复用 blog 的 cdp-browser-publisher 思路）
  - 截图 / 屏幕 OCR（复用 blog 的 ddddocr）
- **F-CLI-6** Agent Loop：调用 LLM → 解析 tool_call → 执行 → 反馈 → 循环
- **F-CLI-7** 积分余额显示（顶栏小角标，实时更新）
- **F-CLI-8** 设置：模型偏好（在管理员授权的模型里选）、主题、快捷键

### 服务端（FastAPI on blog）
- **F-SRV-1** 用户认证：注册（管理员代办）/ 登录 / 刷新 token / 注销
- **F-SRV-2** 用户信息：当前积分 / 可用模型 / 有效期
- **F-SRV-3** LLM 代理：OpenAI 兼容协议 `/v1/chat/completions`、`/v1/models`
  - 鉴权 + 校验用户额度 + 校验模型权限
  - 转发到上游（方舟 / OpenAI / Claude / 通义 / DeepSeek …）
  - 按 token 计费扣减积分
- **F-SRV-4** 使用日志：记录每次请求（用户、模型、tokens in/out、积分、时间戳）
- **F-SRV-5** 管理员 API（仅 admin 角色可用）：
  - 用户 CRUD（增删改查、批量改密码、批量充值）
  - 上游模型 CRUD（添加 / 启用 / 禁用 / 改 Key）
  - 用户-模型授权矩阵
  - 使用统计（按用户 / 按模型 / 按时段）

### 管理后台（Refine / React Admin）
- **F-ADM-1** 管理员登录（独立账号体系，与普通用户隔离）
- **F-ADM-2** 用户管理：列表 / 创建 / 编辑 / 充值积分 / 改密 / 启停
- **F-ADM-3** 模型管理：上游 Key 配置、定价（每千 tokens 多少积分）
- **F-ADM-4** 授权矩阵：勾选哪些用户可以用哪些模型
- **F-ADM-5** 用量看板：积分消耗趋势、Top 用户、Top 模型
- **F-ADM-6** 操作日志：所有 admin 动作可追溯

## 四、积分与计费规则

### 兑换规则
- **1 积分 = 1000 tokens**（默认）
- 每个模型可有自己的"倍率"，例如：
  - 方舟 doubao：×1（即 1000 tokens 扣 1 积分）
  - GPT-4o：×3（即 1000 tokens 扣 3 积分，因为贵）
  - Claude 3.5 Sonnet：×3
- 调用结束后 `actual_credit = ceil((prompt_tokens + completion_tokens) / 1000 × multiplier)`
- 余额不足时拒绝调用（HTTP 402 Payment Required）

### 用户视角
- 工作台顶栏显示"剩余积分：488"
- 单次请求开始前 → 服务端预估上限 → 不够直接拦截
- 长任务（Agent Loop 多轮）会持续扣减，可在设置里设"单任务上限"以防失控

## 六、非功能性要求

| 项 | 目标 |
|----|------|
| 客户端启动时间 | ≤ 3s（Tauri 实测都能做到） |
| 聊天首字延迟 | ≤ 1.5s（上游限制为主，本地不增加） |
| 同时在线用户 | 100+（blog 单机 FastAPI + uvicorn 足够） |
| 安装包大小 | ≤ 20 MB |
| 客户端内存占用 | ≤ 200 MB（idle） |
| 服务端可用性 | 99.5%（一台 blog 单点，能保证就行） |

## 七、安全 & 隐私

- 用户密码 **bcrypt** 加盐哈希存储
- JWT 用 HS256 + 1 小时过期；Refresh Token 30 天
- 上游 LLM API Key **永不下发到客户端**，仅服务端持有
- 客户端本地文件读写 **不上传服务端**，仅在 LLM 上下文里摘要使用
- 高危操作（删除文件、写注册表、执行任意命令）→ 客户端**审批弹窗**确认
- HTTPS only（blog 已配 SSL）
- 管理后台独立认证体系 + IP 白名单可选

## 十、风险 & 待办

| 风险 | 应对 |
|------|------|
| 用户滥用刷 token | 单用户 QPS 限流 + 单任务积分上限 |
| 上游 LLM 抖动 | 多渠道 failover（同一模型可配多个上游 Key） |
| 客户端本地命令执行被恶意 prompt 诱导 | 高危操作审批门 + 黑名单（如 format / del C:\\） |
| blog 单点 | 后期可上 Docker + 备份脚本 |
| 用户 100+ 时 SQLite 写入瓶颈 | 监控触发后迁 PostgreSQL，schema 兼容 |
