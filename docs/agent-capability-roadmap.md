# YT 桌面版 Agent 能力对齐路线（对标 OpenClaw）

> 目标：把 YT 桌面版从「套壳云端对话」逐步做成「能自我判断、自我扩展的本地 Agent」，尽量对齐 OpenClaw（初号机）的能力。
> 原则：慢工出细活，一步一步来。每个阶段可独立交付、独立验证，不追求一次到位。

---

## 一、先搞清楚：OpenClaw「聪明」到底靠什么

读了 OpenClaw 源码文档（agent-loop / context-engine / memory-builtin / system-prompt），它的能力不是单点，而是**六层能力叠加**：

| # | 能力层 | OpenClaw 怎么做 | 作用（为什么显得聪明） |
|---|--------|----------------|----------------------|
| 1 | **强 Agent Loop** | 单会话串行 loop：intake→组装上下文→推理→工具→流式回复→持久化；带队列、超时、重试、compaction 触发重试 | 多步任务不断线、能自我驱动到完成 |
| 2 | **丰富工具集** | read/write/edit/apply_patch/exec/process/web/browser/canvas/纸面一大堆，且**分层审批**（exec approvals / 高危闸门） | 手脚多，能真正动手干活而非只会说 |
| 3 | **技能系统 (Skills)** | `SKILL.md` 声明式技能，按需 `read` 加载；不占常驻上下文；可热更新 | 遇到专门任务能"临时学会"标准流程 |
| 4 | **上下文引擎 + Compaction** | 每轮 assemble 出预算内的消息集；超窗自动摘要压缩；可插拔引擎 | 长对话不爆窗、旧信息不丢主线 |
| 5 | **记忆系统 (Memory)** | MEMORY.md + memory/*.md 索引进 SQLite，FTS5 关键词 + 向量 + 混合检索；跨会话召回 | 记得住偏好/历史/决策，越用越懂你 |
| 6 | **系统提示 + 人格 (SOUL/AGENTS)** | 分层 prompt：base + skills + bootstrap 上下文 + 人格文件 | 稳定的行为准则、身份、边界 |

**关键洞察**：YT 现在只有第 2 层的一小部分（几个工具）+ 第 1 层的雏形（8→16 轮 loop）。差距最大的是 **3/4/5 层**——技能、上下文管理、记忆。这三层才是"自我判断、越用越强"的根。

## 二、YT 现状 vs 目标（诚实对照）

| 能力层 | OpenClaw | YT 现状（v0.2.0） | 差距 |
|--------|----------|------------------|------|
| Agent Loop | 队列/超时/重试/自动 compaction | 非流式，16 轮上限，无重试无压缩 | 大 |
| 工具集 | 数十种 + 分层审批 | read/write/edit/list/glob/grep/shell + 云端 calc/search/fetch | 中 |
| 插件扩展 | 完整插件系统 + 热加载 | ✅ 已做（~/.yt/plugins 热插拔） | 小（已追平雏形）|
| 技能系统 | SKILL.md 按需加载 | ❌ 无 | 大 |
| 上下文/压缩 | 上下文引擎 + compaction | ❌ 无，历史全量塞或截断 | 大 |
| 记忆 | SQLite + FTS/向量/混合检索 | ❌ 无，每次对话空白 | 大 |
| 人格/准则 | SOUL/AGENTS 分层 prompt | 仅一个 system_prompt 设置项 | 中 |
| 流式输出 | 有 | ❌ 非流式（等全部完成才显示）| 中 |

## 三、路线图（分阶段，一步一步来）

> 每阶段末尾标注：**要不要重新编译 Rust**（决定是否走 GitHub Actions）。前端 TS 改动免编译（重新 build dist scp 到 blog 即可，网页版立即生效；桌面版需重新出安装包，但逻辑同款）。

### ✅ M8（已完成）— 插件系统 + 工具扩充
热插拔插件 + grep/glob/edit_file，loop 16 轮。**这是地基。**

### ✅ M9（已完成，网页版已上线）— 流式输出 + Agent Loop 强化
让对话"边想边出字"，并让 loop 更像 OpenClaw：
- ✅ **流式**：Agent Loop 改流式 SSE，逐字显示；分片 tool_calls 按 index 拼接（已验证跑通）
- ✅ **失败自纠错**：工具报错/被拒/未知工具时把提示塞回让模型自己重试换法，不直接终止
- ✅ **行为准则注入**：system prompt 加 Agent 工作方式引导（拆步骤/用工具/自纠错/达成即停）
- ✅ **轮次软提示**：接近上限时提示模型收敛
- 验证：登录→流式带 tools→calculate 多轮全链路通过（临时账号已清理）

### 🔜 M10 — 本地记忆系统（前端+后端，免编译核心，可选 Rust 加速）
对标 OpenClaw memory：
- **记忆文件**：`~/.yt/memory/MEMORY.md` + `~/.yt/memory/YYYY-MM-DD.md`
- **写入工具**：`memory_write`（Agent 可主动记事）
- **检索工具**：`memory_search`（先做 FTS 关键词，够用；向量检索后置）
- **自动召回**：每次对话前，把相关记忆片段注入上下文（system prompt addition）
- 产出：YT「记得住」，跨对话连续。这是"越用越懂你"的关键。

### 🔜 M11 — 技能系统（对标 Skills，插件系统已铺好路）
- `~/.yt/skills/<名>/SKILL.md`：声明式技能（描述 + 触发条件 + 步骤）
- 启动时只加载技能"目录"（名+描述），不占上下文
- Agent 判断任务匹配某技能 → 按需读取完整 SKILL.md → 照做
- 复用 M8 插件的目录扫描机制，很自然
- 产出：YT 能"临时学会"标准流程（写报告、查征信…），像我用 skill 一样

### 🔜 M12 — 上下文管理 + Compaction（前端逻辑）
- token 预算估算，超窗自动摘要旧消息（调模型做 summary）
- 保留最近 N 轮原文 + 老历史压成摘要
- 产出：长对话不爆、不失忆

### 🔜 M13 — 人格/准则分层（对标 SOUL/AGENTS）
- 本地 `~/.yt/AGENTS.md`（行为准则）+ `~/.yt/SOUL.md`（人格）
- 组装进 system prompt，用户可编辑
- 产出：稳定人格与边界，可定制

### 后续可选
- 向量检索（embedding，需后端支持）
- 浏览器控制工具（对标 browser，桌面端可行）
- 子任务/多 Agent（对标 subagents）

## 四、执行约定

1. **前端优先**：M9/M12/M13 主要是前端 TS，改完 build→scp 即可验证（网页版秒生效），桌面版再出包。省去等云端编译。
2. **Rust 只在必要时动**：M10 记忆若用纯文件+前端逻辑可免编译；要本地 SQLite/向量加速才动 Rust（攒一批一起编译）。
3. **每阶段独立验证**：先在网页版（同源 /yt-api）验证逻辑，再打桌面版 tag 出安装包。
4. **Token 节流**：大改由我（OpenClaw）产出架构 + 关键代码；重复 UI/CRUD 可下放本地 IDE。
